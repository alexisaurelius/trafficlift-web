Fix admin upload "Request failed" by removing API redirects on auth failure

The PATCH from the admin audit upload form was hitting a 307 redirect to
/sign-in (caused by Clerk middleware and the legacy redirect() inside
requireUserRecord). The browser then followed the 307 with the PATCH
body, hit /sign-in, and tried to JSON.parse the HTML response, which
threw and surfaced as "Request failed."

- proxy.ts: return JSON 401 for protected /api/* routes when the user is
  unauthenticated instead of redirecting; pages keep the sign-in redirect.
- auth-user.ts: split into requireUserRecord (page redirect) and
  requireUserRecordOrThrow (throws "Unauthorized" for API handlers); same
  for the admin variants.
- Switch all /api/* route handlers to the throwing variants so they map
  to clean 401 JSON responses instead of triggering NEXT_REDIRECT.
- admin-audits-panel: surface real server error text (parses JSON when
  available, falls back to status code or short body) so future failures
  show their actual cause instead of a generic "Request failed.".
